# Captain's Log — 2026-03-06
Launched two new projects today, which always means more time debugging infrastructure than writing actual features.

The bigger chunk of the day went into bootstrapping captains-log — the system that generates these very entries. The initial commit was substantial: 48 files, covering the core log generation logic, GitHub Actions workflows for nightly runs, and the accumulated log backlog. Getting the automation right took several iterations. The cron schedule needed fixing (it wasn't firing at the right time — midnight UTC, not whenever the runner felt like it), the commit message format in the workflow was malformed, and I had to rename `GITHUB_TOKEN` to `PERSONAL_GITHUB_TOKEN` throughout the config and workflow files. That last one is a common friction point — GitHub's default `GITHUB_TOKEN` has restricted scopes by default, so if you need the workflow to push commits back to the repo, you need a PAT with explicit write access, and the naming convention needs to be explicit to avoid confusion. Three separate fix commits for what should have been a clean initial setup, but that's the cost of writing automation you can't easily test locally before it hits CI.

The other project was a LinkedIn profile optimizer — a script that presumably pulls profile sections and runs them through some kind of rewriting or scoring logic. Useful for anyone who wants to treat their professional presence more systematically. The more interesting cleanup work there was removing accidentally committed output folders from git tracking. Classic mistake: generate some local output, forget to add it to `.gitignore` before the first commit, and suddenly you're tracking a bunch of generated files you never wanted in version control. The fix is mechanical — remove from tracking with `git rm --cached`, update `.gitignore` — but it's the kind of thing that's worth catching early before the output folders accumulate hundreds of files across dozens of commits.

Meta observation: it's a little strange to be debugging the system that writes these logs while the logs themselves are being generated. The cron fixes in particular — I was essentially correcting the schedule of a bot that's narrating my own work. There's something recursive about that I haven't fully decided how I feel about.
