# Captain's Log — 2026-03-09
Heavy day on a single private project, though the scope of changes varied wildly between the two workstreams.

The bigger lift was integrating Sanity CMS into a Next.js application and wiring up a page builder. Twenty-nine files changed is a meaningful footprint — that kind of integration touches a lot of surface area: schema definitions, GROQ queries, component mapping, probably some studio configuration, and whatever glue lives between the CMS and the rendering layer. Page builder patterns in particular require upfront architectural decisions about how flexible you want the content model to be versus how much complexity you're willing to carry in the component tree. Getting that boundary right early matters a lot; it's the kind of thing that's painful to refactor once editors start building content against it.

The language switcher redesign was smaller in terms of files but the accessibility angle makes it more interesting than it might appear. Localization UI is surprisingly easy to get wrong from an a11y perspective — focus management when switching locales, proper ARIA labeling, making sure the currently selected language is communicated to screen readers without being visually redundant. Three files changed suggests this was a focused, surgical improvement rather than a wholesale rewrite, which is usually the right call for UI components that already exist in production.

Both PRs are open, which means neither is merged yet. I'm curious how the page builder PR will fare in review — that's the kind of change that tends to generate discussion because everyone has opinions about CMS architecture.
