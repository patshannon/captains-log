# Captain's Log — 2026-03-19
All of today's energy went into a single private project, but it was a productive one — three meaningful workstreams landed across UI, auth, and infrastructure.

The biggest visual win was the Home Hero section merging into develop. That branch had been running parallel to develop long enough that the merge commit touched 91 files — always a moment of mild dread, but it went cleanly. The hero is the first thing users see, so getting that right matters. Alongside it, I shipped a redesigned language switcher with a proper accessibility overhaul. The previous implementation was functional but rough around the edges — the new version uses the object form for `router.replace` rather than a string, which is the right call in Next.js when you need to preserve query params and avoid subtle navigation bugs. The a11y improvements were overdue: language selection is exactly the kind of interactive element that needs correct ARIA roles and keyboard nav.

On the auth side, Facebook SSO is now opened as a PR. Implementing social login is one of those features that looks simple from the outside but touches a surprising amount of surface area — OAuth flow, session handling, account linking if the email already exists through another provider. Eight files changed, which tracks. I'd expect a few more before it merges.

The deployment fix was small but important: Cloud Run wasn't receiving `GOOGLE_CLOUD_PROJECT` as an environment variable, which would cause anything that relies on implicit GCP project resolution to fail at runtime rather than loudly at startup. One-line fix, easy to miss in review, painful in production.

The translation import fix also needed sorting — a broken build on the hero branch was blocking progress there, so that got patched quickly before the merge. Nothing glamorous, but keeping CI green is part of the job.
