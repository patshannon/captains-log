# Captain's Log — 2026-03-17
Heavy deployment day. Most of the energy went into private-project-2, which got a meaningful infrastructure overhaul alongside a new access control feature — then was pushed all the way through to production.

The biggest structural change was migrating CI/CD away from Vercel and over to Firebase Hosting. Vercel is convenient for front-end deploys but can get expensive or limiting depending on usage patterns, and Firebase Hosting integrates more naturally when the rest of the stack is already in the Firebase ecosystem. The migration touched 18 files — workflows, config, environment setup — and the new pipeline threads through develop, staging, and main in a proper promotion model. As part of that, I also introduced an environment variable for the web components URL rather than hardcoding it, which gives us the flexibility to point different environments at different builds of those components without touching application code. That flexibility proved immediately useful: there was a quick follow-up fix to correct the web components URL specifically for the production deployment, which got its own PR and merged same-day.

The other notable addition on private-project-2 was gating dashboard access to partner users only. Blocking unauthorized access at the routing or middleware level rather than just hiding UI elements is the right call — it's a one-line guard that prevents a whole class of access control bugs down the road.

All of this moved through the full develop → staging → main flow over the course of the day, with several rounds of PRs opened and merged in fairly quick succession. That kind of rapid promotion cadence is only comfortable when you trust the pipeline, so it's satisfying that the new Firebase setup held up on its first real run.
