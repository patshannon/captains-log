# Captain's Log — 2026-03-13
Two distinct workstreams today, which is pretty typical for a Friday-ish mid-week split between side project work and client obligations.

The bulk of the interesting work happened in linkedin-bot. The big one was a full rethink of `matcher.py` — I ripped out the old keyword weighting system and replaced it with AI-powered job matching. The old approach was brittle in the way all keyword systems are: too literal, too sensitive to phrasing variations, and constantly needing manual tuning when job postings used synonyms or unconventional language. Moving to an AI-driven scoring mechanism means the matcher can reason about semantic similarity rather than counting term frequency. The tradeoff is obviously latency and cost per match, but for this use case — where precision matters more than throughput — that's an acceptable trade. I also made a small but meaningful adjustment to how the system handles hybrid roles, which was a gap in the context passed to the model. Remote vs. hybrid vs. onsite has become a real filter for most people, so getting that right in the matching context matters.

The other push today was a fix in a private project — the Google SSO work I've been chipping away at. The issue was straightforward: the test suite for `SsoButton` was failing because `setCustomParameters` on the Firebase auth provider wasn't mocked. It's one of those things that's mildly annoying in isolation but actually signals something worth paying attention to — the test boundary wasn't well defined, and the component was reaching further into Firebase internals than it probably should. The fix was quick, but it flagged a refactor worth doing later. Opened the PR for the broader SSO feature (LVC-201) so it can move into review.

Not a massive day in terms of volume, but the matcher refactor is one of those changes that fundamentally shifts what a system is capable of. Worth taking a moment to appreciate that.
