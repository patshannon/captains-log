# Captain's Log — 2026-03-05
Two separate threads today, both converging on the same underlying theme: infrastructure that quietly breaks when you forget to thread a variable through properly.

The issue was straightforward but annoying — Sanity's configuration requires certain values to be baked in at build time, and the Dockerfile was missing the corresponding `ARG` declarations. Without those, Docker silently ignores the `--build-arg` flags passed during the build, meaning the environment variables never made it into the image. Fixed the Dockerfile first, then realized the CI workflow had the same gap — the build step wasn't forwarding those values either. Two commits for what should have been one, but that's how it goes when you find the second hole only after plugging the first.

The other thread was more interesting: laying the groundwork for a LinkedIn job application automation tool. The base resume generator script I added today handles configuration loading and output options — the scaffolding that everything else will depend on. The core idea is to generate tailored resume variants programmatically rather than manually tweaking a document each time. Getting the config and output abstraction right at this stage matters a lot, because it's the seam between the static resume content and whatever dynamic targeting logic comes later.

Speaking of resume content — synced the public resume repository with an updated base. Four files changed, which usually means the content update rippled across multiple output formats or sections. Keeping the source of truth in one place and generating from it is the right call; I've been burned too many times by diverging copies.
