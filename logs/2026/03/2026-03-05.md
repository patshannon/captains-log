# Captain's Log — 2026-03-05
Scattered but productive day — a good mix of infrastructure fixes, feature work, and tooling.

Most of the morning was spent untangling a Docker build issue on a private project. The Dockerfile was missing ARG declarations for Sanity-specific configuration values, which meant they were silently dropped during the build rather than passed through properly. That kind of bug is annoying because it fails quietly — the build succeeds, the app just doesn't behave correctly at runtime. Fixed the Dockerfile itself and patched the CI build workflow to ensure those environment variables are actually present when the image gets assembled. Small changes, but the kind of thing that blocks everyone until it's done.

After that, I started laying out the home hero section on the same project — five files touched, so there's some meaningful structure coming together there. Opened a PR for it; it's early but the scaffolding is in place.

On two other private projects, there was a shared thread of work around a "Type de suivi" field — adding it to a caseload export on one side, and updating the display logic and checkbox label on the other. These two projects appear to share domain concepts, so keeping them in sync matters. The section 2 revision PR merged cleanly with 26 files changed, which suggests a meaningful chunk of UI or data model work landed today.

The more interesting side project: I added a base resume generator script to `linkedin-bot`. This fed directly into the `resume` repo, where a Claude-assisted PR updated the actual resume content from the new base, and a second PR opened to improve the about page copy for clarity and voice. The automation pipeline is starting to close the loop: generate → update → refine.

Eight commits, five PRs, four different repos. Not a focused day in terms of depth, but a lot of threads moved forward.
