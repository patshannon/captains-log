# Captain's Log — 2026-03-18
Heavy deployment day. Three projects, a steady stream of PRs, and more branch promotions than I care to count — the kind of day where you spend as much time shepherding releases as writing code.

The bulk of the meaningful work landed in private-project-2 and private-project-3, both related to the same feature arc: a "contributions" flow for a professional-facing portal. The feature itself is clearly not quite ready — the approach for now was to surgically remove the creation entry point from the navigation and wire up a route guard to block direct access. Hiding behind a guard rather than a flag or a feature toggle is a pragmatic short-term call; it keeps the code in place and deployable without exposing an incomplete flow to users. The SSO sign-in path also got a `redirectTo` parameter threaded through it, which is likely groundwork for sending pro users back to where they were trying to go after authenticating — a small quality-of-life improvement that's easy to miss but annoying when it's absent.

The hotfix in private-project-2 was more urgent: event and exhibition search results were surfacing unpublished content. That's the kind of data-visibility bug that can't sit in a queue, so it got its own PR, merged fast, and pushed straight to prod. The pre-merge cleanup commit that went alongside the larger contributions cherry-pick — restoring exhibition search, stripping debug logs, fixing typos — suggests this branch had a rougher journey than usual before it landed.

Over in private-project-3, the web components URL got corrected from a staging/dev endpoint to the production one before deployment. Easy to miss, painful if you don't catch it. The partner login guard also got a polish pass.

private-project-1 was purely operational: a data update to a solicitor list for an upcoming event, with a casing fix caught shortly after. Small, but the kind of thing someone would have noticed immediately in a printed program or a formal email.

All told, a coordination-heavy day. Lots of moving parts kept in sync across environments, and nothing caught fire in production. That counts as a win.
