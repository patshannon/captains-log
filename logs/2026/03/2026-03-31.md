# Captain's Log — 2026-03-31
Day 90, and it was split pretty cleanly between UI work and some fiddly state-preservation bugs in a couple of backend-adjacent workflows.

The UI side was straightforward and satisfying: built out a reusable tabs component for one private project and opened a PR for it. Tabs seem simple on the surface but there's always a decision to make about how much control to hand to the consumer — controlled vs. uncontrolled state, how to handle active tab persistence, accessibility concerns around keyboard navigation and ARIA roles. Five files changed suggests this landed with tests or stories alongside the component itself, which is the right way to ship something meant to be reused.

The heavier cognitive load today came from a cluster of metadata preservation bugs across two other private projects, both touching job offer lifecycle flows. The pattern was the same in both cases: some piece of publish metadata — likely timestamps, scheduling config, or author attribution — was getting dropped when an offer transitioned state. In one project that meant the renew and reopen flows weren't carrying state through correctly (ten files touched, so this was more than a surgical fix — probably involved rethinking how the state machine hands off data between transitions). In the other, it was specifically scheduled publish metadata being lost, and separately, metadata not surviving the moderation pass.

These kinds of bugs are easy to miss until something operational breaks — a job that was supposed to go live at a scheduled time silently loses its schedule, or an offer that went through moderation comes out the other side missing context. The root cause is almost always the same: a transformation or copy step somewhere in the pipeline that was written to handle the happy path and implicitly drops fields it doesn't know about. The fix is usually defensive — either explicitly threading the fields through or switching to a merge-rather-than-replace pattern.

Solid, grounded day. Nothing glamorous, but the kind of work that keeps systems honest.
