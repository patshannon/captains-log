Two files, quick diff, done.

On a second private project, I added a TTL to a rewrite rule for a webcomponents `index.js` file. Small change, important reason: without a TTL, CDN or browser caches can hold onto stale JavaScript indefinitely, which is particularly painful for web components since they register themselves in the custom element registry — a process that doesn't tolerate duplicate registrations. Clients end up running mismatched versions with no clean recovery path short of a hard refresh. A proper TTL gives us control over cache invalidation and makes deploys predictably safe. One file changed, but it's the kind of one-liner that prevents a class of silent production bugs.

Light day overall, but the work was solid and purposeful.
