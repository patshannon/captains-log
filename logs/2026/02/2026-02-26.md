# Captain's Log — 2026-02-26
One commit today, but a meaningful one. Spent time fixing a subtle CSRF validation issue in a private project where Server Actions were rejecting requests from custom domains — the origin check was too narrowly scoped and didn't account for domains outside the default configuration.

Server Actions in modern React frameworks do their own CSRF mitigation by validating the `Origin` header against a known list of trusted hosts. The problem with a rigid implementation is that it breaks down the moment you introduce custom domains — vanity URLs, tenant-specific subdomains, that sort of thing. The fix was surgical: one file changed, likely threading a custom domain allowlist into the origin comparison logic so legitimate requests from those domains aren't caught in the same net as actual cross-site attacks.

It's the kind of bug that only surfaces in production-like environments where custom domains are actually in play, which is probably why it slipped through. Easy to overlook in local dev where everything runs on localhost. Worth making sure the fix is robust here — CSRF checks are security-critical, so the solution can't just be "widen the check and hope for the best." The right answer is an explicit allowlist, not a looser heuristic.

Light day, but security correctness is never trivial work.
