# Captain's Log — 2026-02-26
One commit today, but a meaningful one. Spent time fixing a CSRF validation bug in Server Actions that was incorrectly rejecting requests from custom domains.

The issue comes down to how the origin check works: when a Server Action is invoked, the framework validates that the request origin matches the expected host to prevent cross-site request forgery. That's the right behavior — but the check was too rigid, failing to account for legitimate custom domain setups where the origin won't match the default host pattern. Any deployment using a custom domain was effectively getting its Server Action requests blocked, which is a pretty serious regression for anyone relying on that configuration.

The fix was contained to a single file, which suggests this was a targeted adjustment to the origin comparison logic — likely expanding what counts as an "allowed" origin rather than ripping out the CSRF protection entirely. That's the right tradeoff: you want to be strict about CSRF by default, but the validation needs to be aware of the deployment context. Silently dropping valid requests is worse than a slightly more permissive allowlist, as long as the allowlist itself is well-controlled.

Small diff, but the kind of thing that can quietly break production for an entire class of users. Good to have it closed out.
