# Captain's Log — 2026-02-04
Routing work dominated today, split across two private projects.

The bulk of the effort went into a web infrastructure change — adding rewrite rules across development, production, and staging environments to properly route web components. This kind of change is deceptively simple on the surface but requires careful attention to environment parity. Getting rewrites wrong in one environment and not another is exactly how you end up chasing phantom bugs that only reproduce in staging. Three config files touched, one for each environment, which suggests these are kept separate rather than composed — worth revisiting at some point, but not today's problem.

Also on that same project: dynamic routes for a bilingual feature, French and English, tied to what looks like a Quebec anniversary campaign. Internationalized routing always has a few subtle traps — slug collisions, fallback behavior, canonical URL handling — so it's good to get this scaffolded early and stress-tested before it matters.

The second project had a more surgical fix. Somewhere along the way, Algolia index name configuration was being pulled from `import.meta.env` — the Vite-native way to access environment variables — but apparently the codebase has since standardized on a shared config object for that kind of thing. Swapping it out is a small change but an important one for consistency: if the index name is ever overridden or abstracted for multi-environment support, you want it flowing through one place, not scattered across direct env reads. Three files touched suggests it wasn't quite as isolated as one would hope, but at least it's clean now.

Relatively focused day — two infrastructure-adjacent tasks and one tidy bug fix.
