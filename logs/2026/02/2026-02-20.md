# Captain's Log — 2026-02-20
Spent most of today in a single private project, but the scope of work varied a lot between the two commits.

The smaller fix was a test repair on an AuthButton component — async behavior and href assertions have a way of quietly breaking when the underlying auth flow changes, and that's exactly what happened here. Nothing exotic, just the kind of test maintenance that accumulates when you're moving fast. Got it back to green.

The bulk of the day went into building out a `UserDataProvider` — a context provider that hooks into Firestore to surface user profile data throughout the app. Nineteen files changed is a meaningful footprint, which tells me this wasn't just a wrapper around a single `useDocument` call. This kind of feature tends to touch a lot of surface area: the provider itself, the context definition, consumer hooks, type definitions, and then everywhere in the component tree that needs to start consuming the new data. There's also the Firestore subscription lifecycle to think carefully about — making sure you're unsubscribing on unmount, handling loading and error states cleanly, and not triggering unnecessary re-renders when profile data hasn't actually changed.

The decision to centralize user profile data in a provider rather than fetching it ad hoc in individual components is the right call at this stage. It keeps Firestore reads consolidated, makes the data shape easy to evolve in one place, and gives you a clean seam for mocking in tests. The tradeoff is that you're committing to a particular data-fetching architecture fairly early — but given this is already PR #8, the project is past the throwaway prototype phase, so that kind of structural investment makes sense now.

Solid day overall. The foundation for user-aware UI is in place.
