# Captain's Log — 2026-04-08
Spent today shipping a design system feature that's been sitting in the backlog — badges and tags. Small surface area on paper, but these foundational UI components tend to surface interesting type system questions once you get into the weeds.

The bulk of the work was in PR #42: getting the Badge and Tag components built out correctly, covering the visual variants, sizing, and making sure they compose cleanly with the rest of the design system. Six files changed for what sounds like "just a couple of components" is about right once you factor in stories, tests, and any shared token updates.

The follow-up commit is the interesting one: widening the `TagAsLink` href type to accept arbitrary strings rather than a more constrained type. This is a classic tension in typed component APIs. The initial instinct is always to be restrictive — accept only route-shaped strings, or a union of known paths — but that bites you the moment a consumer needs to link out to an external URL or a dynamically constructed path. The fix is small (single file) but it's the kind of thing that matters for adoption; a component that forces workarounds will just get bypassed.

Merged into develop, which is the right call for design system work that other features will likely depend on. The captains-log push is just housekeeping — keeping this log repository current.

Ninety-eight days in and it's satisfying to be laying down infrastructure that other work will build on. Design systems are slow ROI, but the compounding effect eventually shows.
