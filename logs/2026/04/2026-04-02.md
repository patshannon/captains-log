# Captain's Log — 2026-04-02
Heavy Sanity CMS day. Almost all the work was in one private project, building out the content schema and page builder blocks that editors will actually use — the kind of work that's unglamorous but absolutely load-bearing.

The most substantial piece was the event collection schema, which landed via PR after a review cycle. The interesting design decision there was supporting two distinct selection modes: filter-based (dynamic, rule-driven) and manual (handpicked). That's a common editorial tension — marketing wants "show me the next 5 upcoming events automatically" while someone else wants to curate a specific set regardless of date logic. Encoding both modes into the schema rather than forcing editors to choose one pattern permanently felt like the right call. Twenty-two files changed, which tells you there's real surface area here — probably schema definition, frontend rendering, and whatever glue connects the two.

The other blocks were smaller but meaningful additions to the page builder. A `ContestBanner` block with an image field, a `completeProfileCta` block for nudging users toward profile completion, and a conditional image section that hides itself when no illustration is set in Sanity. That last one is a detail I appreciate — too many CMS-driven components render awkward empty space because nobody guarded against the null case. Hiding the section entirely when there's no asset is just cleaner.

Also fixed a typing issue on a CTA button link in the CMS. Small thing, but type errors in content schemas have a way of causing subtle bugs at render time when the shape of the data doesn't match what the frontend expects.

Rounded out the day with a Sanity stack upgrade — dependency bumps across four files, probably pulling in Studio v3 improvements or fixes. Not exciting, but staying current with Sanity's release cadence is worth the occasional maintenance tax.
